## 主要功能

1. **统一界面设计**
   - 所有功能在一个界面中，无需切换标签页
   - 顶部：输入区域（ShareID/URL输入、读取文件列表、开始下载、停止下载）
   - 中间：文件列表（显示所有文件信息，支持右键菜单）
   - 底部：日志记录（显示详细日志）
   - 底部：进度信息（总进度条、统计信息、操作按钮）
2. **右键菜单功能**
   - 单独下载：右键点击文件，选择"单独下载"下载单个文件
   - 批量下载：右键点击文件，选择"批量下载"下载多个文件
   - 灵活选择：支持多选文件进行操作
3. **文件信息显示**
   - 文件类型：WORD、PDF、图片、视频、表格、PPT、压缩包等
   - 文件名：完整的文件名
   - 状态：待下载、下载中、已完成、已存在、下载失败
   - 进度：实时进度条显示
   - 速度：实时下载速度统计
   - 文件大小：自动格式化为B/KB/MB/GB
   - 权限：可查看、可下载、完全访问
   - 耗时：下载耗时统计
   - 下载链接：完整的下载URL
4. **下载速度优化**
   - 8KB缓冲区，提高下载速度
   - 禁用HTTP缓存，减少重复下载
   - 实时速度计算，准确显示下载速度
5. **文件夹命名优化**
   - 使用知识库名称作为文件夹名
   - 使用"知识库\_" + ShareID前8位
   - 避免显示null，确保文件夹名称有效
6. **粘贴下载功能**
   - 支持从剪贴板粘贴文件信息并批量下载
   - 自动识别下载链接
   - 支持多种粘贴格式

## 使用方法

### 启动程序

双击 `start.bat` 文件

### 读取文件列表

1. 在输入框中输入ShareID或URL（每行一个）
2. 点击"读取文件列表"按钮
3. 等待文件列表加载完成
4. 查看文件列表中的所有文件信息

### 下载文件

#### 方法1：批量下载所有文件

1. 点击"开始下载"按钮
2. 等待所有文件下载完成
3. 查看实时下载进度和速度

#### 方法2：单独下载文件

1. 在文件列表中右键点击要下载的文件
2. 选择"单独下载"
3. 等待文件下载完成

#### 方法3：批量下载选定文件

1. 在文件列表中选择多个文件（按住Ctrl或Shift键）
2. 右键点击选中的文件
3. 选择"批量下载"
4. 等待文件下载完成

#### 方法4：粘贴下载

1. 复制文件信息到剪贴板
2. 点击"粘贴并下载"按钮
3. 程序自动识别下载链接并开始下载

### 查看下载进度

- **文件列表**：实时查看每个文件的下载进度、速度、状态
- **总进度条**：查看整体下载进度
- **统计信息**：查看总文件数、已完成数、失败数

### 管理下载

- **打开下载文件夹**：点击"打开下载文件夹"按钮，打开下载目录
- **清空日志**：点击"清空日志"按钮，清空日志记录

## 配置说明

配置文件位于 `config/app.properties`，包含以下配置项：

```properties
# 应用配置
app.name=灵枢IMA智能下载器V2.2

# API配置
api.url=https://ima.qq.com/cgi-bin/knowledge_share_get/get_share_info
api.timeout=30000

# 下载配置
download.dir=./downloads
download.chunk.size=8192
download.retry.count=2
download.thread=100

# UI配置
ui.progress.refresh=1000
ui.list.auto.scroll=true
ui.log.max.lines=500
```

## 常见问题

### 1. 程序无法启动

**解决方案**：

- 确保已安装Java 8或更高版本
- 检查lib文件夹中的依赖库是否完整
- 查看控制台输出的错误信息

### 2. 读取文件列表失败

**解决方案**：

- 检查ShareID是否正确（64位十六进制字符串）
- 检查网络连接是否正常
- 查看控制台输出的调试信息

### 3. 下载失败

**解决方案**：

- 检查下载链接是否有效
- 检查网络连接是否稳定
- 查看日志中的错误信息
- 尝试重新下载失败的文件

### 4. 文件夹显示null

**解决方案**：

- V2.2版本已修复此问题
- 会使用"知识库\_" + ShareID前8位

### 5. 下载速度慢

**解决方案**：

- V2.2版本已优化下载速度
- 使用8KB缓冲区
- 禁用HTTP缓存
- 检查网络带宽是否充足
